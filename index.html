🔬 Auto Quantifier Methodology

1. System Architecture 系統核心架構
  • AutoQuantifier 採用 Serverless 的邊緣運算架構，將繁重的影像處理任務封裝於瀏覽器端執行，導入「主執行緒視圖與背景運算分離」以確保極端資料量下的互動流暢度及資料隱私。

1. Dual-Engine Web Workers 雙引擎邊緣運算架構

  • Transient Decode Worker (瞬態解碼引擎)：專職負責 I/O 解析。動態生成以拆解多通道 TIFF 或非同步多檔佇列，透過 Zero-Copy 將 Float32 矩陣傳回主執行緒後立即強制終止 (Terminate)，實現極端記憶體回收 (Aggressive GC)，徹底消除處理大圖時的 OOM 隱患。
  • Persistent Math Worker (常駐數學引擎)：專注於數學形態學與拓撲掃描 (Rolling Ball, BFS, MSER)。利用 Ping-Pong Buffer 與主執行緒進行零拷貝通訊，確保高頻率互動下的極致流暢度。

2. Canvas Stack 多圖層疊棧視圖

  • imgCanvas：負責底圖與對數映射後的高對比渲染。
  • overlayCanvas：負責幾何向量（Bounding Box、質心與狀態標籤）繪製。
  • uiCanvas：處理高頻率互動（如 ROI 拖曳、準星游標），避免不必要的重繪。

3. Memento Pattern 結構共享歷史狀態管理

  • 採用不可變 (Immutable) 的單一資料流設計，以避免 OOM。
  • 利用展開運算子與 Reference 傳遞，僅備份參數與像素索引，不複製龐大矩陣。
  • 實現 30 Steps 無損歷史回溯 (Undo/Redo)。

2. Core Algorithms 核心演算法與數學模型

1. Spatial Geometric Correction 空間幾何校正

  • 系統不採用單一常數，而是導入左右獨立的分段二階多項式 (Asymmetrical Piecewise Quadratic Curve Fitting)，以精準校正非對稱的電泳微笑效應。
  • 演算法採用逆向空間映射 (Inverse Spatial Mapping) 尋找來源座標，再搭配雙線性插值 (Bilinear Interpolation) 重新分配像素能量，確保總積分量絕對不失真。
Asymmetrical Piecewise Quadratic Warp $\Delta y_{src} = \Delta y_{dest} - k \cdot (x_{norm})^2$

為解決西方墨點法常見的「左右變形不均勻」問題，系統將 X 軸相對於質心 $c_x$ 進行正規化，並根據正負號套用獨立的扭曲常數 ($k_L, k_R$)，兩條獨立的拋物線於中心點完美平滑接合。

Step 1: X-Axis Normalization (座標正規化)
$$x_{norm} = \frac{x_{dest} - c_x}{c_x} \quad , \ x_{norm} \in [-1, 1]$$
Step 2: Piecewise Smile Factor (分段微笑因子決策)
$$k = \begin{cases} k_L & \text{if } x_{norm} < 0 \\ k_R & \text{if } x_{norm} \ge 0 \end{cases}$$
Step 3: Inverse Mapping (逆向映射座標反推)

計算目標像素在原始矩陣中的真實對應偏移量 $\Delta y_{src}$：

$$\Delta y_{src} = (y_{dest} - c_y) - k \cdot x_{norm}^2$$
Bilinear Interpolation: $\quad P_{out} = \text{Bilinear}(x_{src}, \ y_{src})$

2. Downsampled Content-Aware Rolling Ball 動態降採樣滾球演算法

  • 基於數學形態學的 2D 滾球演算法 (Morphological Opening) 扣除背景。
  • 當估算半徑 $r$ 過大時，利用縮放因子 $S$，在降採樣的微型矩陣上估算背景再利用雙線性插值放大，大幅降低運算成本：
$$S = \max\left(1, \lfloor \frac{r}{5} \rfloor\right)$$
Standard 2D Morphological Opening $f \circ B = (f \ominus B) \oplus B$

傳統 2D 滑動視窗的時間複雜度極高 $\mathcal{O}(W \cdot H \cdot r^2)$。系統將 2D 結構元素 $B$ 數學解構，轉換為水平 ($X$) 與垂直 ($Y$) 的 1D 陣列進行四道連續掃描，將時間複雜度極限降維至線性的 $\mathcal{O}(W \cdot H \cdot r)$：

Step 1: 1D Horizontal Erosion (X 軸侵蝕 - 尋找局部極暗點)
$$E_x(x, y) = \min_{k \in [-r, r]} f(x+k, y)$$
Step 2: 1D Vertical Erosion (Y 軸侵蝕 - 補齊 2D 侵蝕)
$$E_{xy}(x, y) = \min_{k \in [-r, r]} E_x(x, y+k)$$
Step 3: 1D Horizontal Dilation (X 軸膨脹 - 恢復波段邊界)
$$D_x(x, y) = \max_{k \in [-r, r]} E_{xy}(x+k, y)$$
Step 4: 1D Vertical Dilation (Y 軸膨脹 - 建立最終背景基線)
$$Bg(x, y) = \max_{k \in [-r, r]} D_x(x, y+k)$$
Final Subtraction: $\quad f_{out}(x, y) = \max(0, f(x, y) - Bg(x, y))$

3. Logarithmic Mapping 對數空間映射

  • 將浮點線性強度非線性地映射至視覺 RGBA 空間以突顯極弱訊號（僅影響渲染，不污染定量數據）。常數 $C$ 確保輸出落於 0~255 區間：
$$V_{out} = C \cdot \ln(1 + V_{in})$$

4. Hysteresis Thresholding & BFS 雙閾值水流蔓延

  • 系統揚棄了傳統極易造成斷層的單一全局閾值 (Global Thresholding)，改採高容錯的雙閾值遲滯掃描 (Hysteresis Thresholding) 配合廣度優先搜尋 (BFS) 進行波段蔓延。
  • 為因應極端弱訊號與高強訊號在視覺上的落差，滑桿與底層真實物理強度的映射加入了非線性 Gamma 權重 ($\gamma$)，確保在低強度區域具備極高的微調解析度。
Non-linear Hysteresis BFS Growth $Seed \rightarrow Expand(I \ge T_{boundary})$

傳統演算法容易因單一閾值過高而丟失弱波段邊緣，或因閾值過低導致相鄰波段相互融合。系統定義了高強度的 Core (種子) 與低強度的 Boundary (擴張底線)，從種子出發向外蔓延，完美兼顧強訊號的定位與弱邊緣的保留。

Step 1: Non-linear Intensity Mapping (非線性物理強度映射)

將 UI 滑桿百分比 $P_{ui}$ 透過 $\gamma$ (Linearity Scale, 預設 2.5) 映射回真實物理閾值 $T$：

$$T = I_{max} \cdot \left(\frac{P_{ui}}{100}\right)^{\frac{1}{\gamma}}$$
Step 2: Seed Initialization (核心種子篩選)

尋找影像中強度大於等於 Core 閾值的所有像素作為水流源頭：

$$S = \{ p \in Image \mid I(p) \ge T_{core} \}$$
Step 3: BFS Hysteresis Expansion (雙閾值水流蔓延)

從種子 $S$ 出發，向外遞迴擴張至周邊 8 鄰域像素 $p_{adj}$，直到強度低於 Boundary：

$$Expand(p_{adj}) \iff I(p_{adj}) \ge T_{boundary}$$
※ Automated Statistical Guess (動態起始邊界估算)

首次點擊分析時，系統自動擷取有效像素之平均值 $\mu$ 與標準差 $\sigma$，推算統計學最佳起始滑桿位置：

$$T_{auto\_start} = \mu + 0.5\sigma$$

5. 1D Intensity-Projected Watershed 訊號降維切割

  • 針對肉眼可見但傳統演算法難以切分的沾黏波段 (Fusion Blobs)，系統引入降維地形學，將 2D 強度沿 Y 軸投影為 1D 光譜進行降噪與波谷切割。
  • 切割後「🪄 Magic Wand」套用專屬的強度對稱裁切 (Intensity-Symmetric Trimming) 技術，完美解決分水嶺切割後，朝向背景端產生過度蔓延的不對稱拖尾現象。
1D Spectral Projection & Symmetric Trimming $Trim \iff I_{profile}(y) \ge T_{sym}$
Step 1: 1D Spectral Projection & Smoothing (一維光譜投影與降噪)

面對沾黏波段 (Blob Fusion)，沿 Y 軸將 2D 像素群投影為 1D 光譜 $P(y)$，並套用高斯平滑核 (Gaussian Kernel) 進行深度降噪：

$$S(y) = \sum_{k=-2}^{2} P(y+k) \cdot W(k+2) \quad \text{where } W = [0.05, 0.25, 0.40, 0.25, 0.05]$$
Step 2: Valley Detection (波谷深度檢測)

尋找 1D 光譜的波谷作為切割線。波谷的相對深度必須嚴格大於相鄰波峰的 $5\%$，才會被判定為有效的分水嶺：

$$\frac{\min(Peak_1, Peak_2) - Valley_{min}}{\min(Peak_1, Peak_2)} > 0.05$$
Step 3: Intensity-Symmetric Trimming (強度對稱裁切)

為解決弱訊號被 Watershed 切割後，朝向背景端產生過度蔓延的不對稱「拖尾」現象。系統抓取波段上下邊緣的最高強度，計算對稱閾值並由波峰向外重新執行精準裁切：

$$T_{sym} = \max(I_{top}, I_{bottom}) \times 0.95$$
$$Trim \iff I_{profile}(y) \ge T_{sym}$$

6. Auto HDR: Maximally Stable Extremal Regions (MSER) 拓撲掃描演算法

  • 系統內建自動化 MSER (最大穩定極值區域) 演算法。透過模擬「水位下降（閾值遞減）」，在多層級的拓撲結構中尋找面積變化率趨近於零的「完美收斂高原期」。
  • 結合主執行緒的狀態機記憶與 Web Worker 的平行運算，達成無需人工干預的 Auto HDR 訊號捕捉。
MSER Topographical Sweep & Phase 3 Rescue $\arg\min \frac{\Delta Area}{Area} \ \ \forall T \in [T_{start}, T_{end}]$

Auto HDR 透過自動迭代雙閾值 BFS 演算法，追蹤每個波段在不同閾值下的拓撲衍生歷史 (Lineage)，以數學導數尋找最穩定的物理邊界。

Step 1: Threshold Descent (水位下降迭代)

從使用者定義的高強度起點向下掃描至雜訊底線，步進值為 $\Delta T$：

$$T_i = T_{start} - i \cdot \Delta T \quad \text{for } T_i \ge T_{end}$$
Step 2: Stability Criterion & Forgiving Cache (穩定度評估與寬容快取狀態機)

計算波段在相鄰閾值間的面積成長率 $\Delta(T_i)$。系統揚棄死守絕對極小值的傳統做法，建立「寬容快取狀態機 (Forgiving State Machine)」，若成長率極低 ($\le 12\%$) 或起伏未偏離歷史最佳狀態太多 ($+8\%$ 內)，即視為達到拓撲高原期 (Plateau)，系統將自動快取此最佳狀態，允許波段貪婪地完美收斂：

$$\Delta(T_i) = \frac{Area(T_{i}) - Area(T_{i-1})}{Area(T_{i-1})}$$
$$Update \ Cache_{plateau} \iff \Delta(T_i) < 0.12 \ \lor \ (\Delta(T_i) \le \Delta_{best} + 0.08 \ \land \ \Delta(T_i) < 0.25)$$
Step 3: Adaptive Morphological Descent (Phase 3 適應性打撈)

針對沉沒於底層雜訊中的極弱訊號，系統會萃取已成功收斂波段的「形態學」($W_{ref}, H_{ref}$)，並套用高張力邊界比例 ($\rho = 0.85$) 進行弱訊號強制剝離，同時避免雜訊沾黏：

$$T_{boundary} = \max(T_{end}, \ T_{core} \times 0.85)$$
$$Rescue \iff (W_{new} \approx W_{ref}) \land (H_{new} \approx H_{ref}) \land \text{No Collision}$$

7. Magic Wand: Log-Driven BFS 魔術棒局部對數空間蔓延

  • 面對極端微弱且被背景雜訊淹沒的訊號，線性水流極易引發全局溢流。系統於主執行緒實作了局部對數空間廣度優先搜尋 (Log-Driven BFS)，以達成零延遲互動。
  • 此模式將線性強度轉化為對數空間以放大暗部特徵，同時結合泳道物理箝制 (Lane Clamping)，確保水流蔓延絕不越界至相鄰泳道。
Log-Driven BFS with Dynamic Lane Clamping $Expand(p) \iff \ln(1+I_{adj}) \ge 0.55 \cdot Seed_{log}$

在極低訊號雜訊比 (SNR) 的情況下，線性閾值往往會失效。 🪄 Magic Wand 模式能動態建立局部泳道限制視窗，將浮點訊號映射至非線性對數空間放大微細特徵，使 BFS 能在充滿雜訊的背景中執行高強度的封閉式探測，最後透過 1D Watershed 與 Intensity-Symmetric Trimming 精準剝離出符合物理特徵的蛋白質訊號。

Step 1: Dynamic Lane Clamping (動態泳道物理鉗制)

提取全域拓撲寬度的中位數 $W_{ref}$，以使用者點擊座標 $x_{click}$ 為中心，建立絕對禁止越界的 X 軸物理防禦牆：

$$X_{lane} = \left[ x_{click} - \frac{W_{ref}}{2}, \ x_{click} + \frac{W_{ref}}{2} \right]$$
Step 2: Seed-Centric Peak Detection (局部對數尋峰)

在受泳道限制的 $5 \times 5$ 微型視窗內，尋找對數空間中的局部極大值 $L_{max}$ 作為水流源頭：

$$Seed_{log} = \max_{p \in W_{5 \times 5}} \ln(1 + I_{linear}(p))$$
Step 3: Log-Driven Expansion (對數驅動蔓延)

從極值點啟動 BFS 水流。僅當相鄰像素的對數強度大於等於極值的 55% 時，才允許擴張，藉此強行剝離漸層光暈：

$$Expand(p_{adj}) \iff \ln(1 + I_{linear}(p_{adj})) \ge 0.55 \cdot L_{max} \ \land \ x_{adj} \in X_{lane}$$

8. Topological QC & Defense Mechanisms 拓撲品管與防禦機制

  • 無論是局部打撈 (Magic Wand) 或全域掃描 (Auto HDR)，水流蔓延後擷取到的物件 (Blobs) 均需通過嚴格的幾何特徵與拓撲審查，以剔除數位雜訊、光暈殘骸與過度分割之偽影。
🪄 Magic Wand Isolation QC $QC_{wand}(Blob) \iff Width \ \land \ Intent \ \land \ Solidity$
Step 1: Dynamic Width Reference (動態寬度基準萃取)

萃取畫面上已建立拓撲之波段寬度的中位數，做為物理寬度的審查基準 $W_{ref}$，並限制水流不可蔓延超出該泳道邊界：

$$W_{ref} = \text{Median}( \{ W_{blob} \mid \forall blob \in Topology \} )$$
$$QC_{width} \iff 0.4 W_{ref} \le W_{wand} \le 1.6 W_{ref}$$
Step 2: Intent Retention (意圖鎖定過濾)

當 1D Watershed 切割連體波段時，僅保留「包含使用者最初點擊座標 $P_{click}$」的子波段，捨棄其餘受牽連之殘骸：

$$QC_{intent} \iff P_{click} \in Blob_{sub}$$
Step 3: Solidity & Shape QC (實心度與像素下限防護)

防止空心的幽靈雜訊或微小灰塵被判定為有效波段：

$$Solidity = \frac{Area_{pixels}}{W_{wand} \times H_{wand}}$$
$$QC_{shape} \iff Solidity \ge 0.05 \ \land \ Area_{pixels} > 8$$
✨ Auto HDR / MSER Topographical Defense $\text{Four-Shield Defense} \ \land \ \text{Global Feature QC}$
Step 1: Parent-Centric Evaluation (四大防護審查)

監控波段在閾值下降時的增長率 $\Delta$。若觸發以下任一溢流條件或實體排斥，系統將立即阻斷擴張並回退至歷史最佳快取：

  • Area Spill (面積暴增)： $\Delta Area > Threshold_{spill}$
  • Width Spill (左右融合)： 實際變寬 $>5px$ 且 $\Delta Width > 70\%$
  • Height Spill (上下吞噬)： 實際變高 $>4px$ 且 $\Delta Height > 40\%$
  • AABB Collision (實體排斥)： 邊界框觸碰已鎖定之波段 (維持 1px 社交距離)
Step 2: Forgiving Plateau Tracking (寬容高原期快取)

不盲目鎖定絕對最低成長率，若 $\Delta Area < 12\%$ 或僅微幅波動，視為安全高原期並持續更新快取，允許波段「貪婪地完美收斂」。

Step 3: Global Feature QC (基於全域母體的中位數特徵過濾)

掃描結束後，統整畫面上所有波段之寬高分佈，推算動態物理上下限，精準剔除偽影：

  • Vertical Artifact (垂直殘影)： 高度 $> \text{寬度} \times 3$
  • Scanner Scratch (掃描器刮痕)： 寬度 $> \text{高度} \times 15$
  • Ghost Shape (空心幽靈雜訊)： 實心度 < 0.02 且面積過小

9. Multiplex IFD & Multi-File Spatial Alignment 多通道與多檔空間對齊

  • 針對現代西方墨點法 (如 Li-COR 雙色螢光) 或不同曝光時間之多檔疊加需求，系統內建非同步解析佇列 (Async Multi-File Queue)，能將多重 IFD 圖層或獨立圖檔疊合成統一的專案通道 (Channels)。
  • 切換通道時觸發實體框線狀態保留 (Persistent ROI State)，確保不同圖層間的定量區域於物理座標上絕對對齊，達成精準的跨通道比值計算。
Multiplex Spatial Alignment $ROI_{CH1}(x,y) \equiv ROI_{CH2}(x,y)$
Step 1: Multiplex IFD Heuristics (多通道特徵啟發式過濾)

自動過濾 TIFF 內的低解析度縮圖，嚴格鎖定最大面積的實體圖層作為有效的螢光通道：

$$Channel_{valid} = \{ ifd \in TIFF \mid Area(ifd) = \max(Area) \}$$
Step 2: Cross-Channel Persistent ROI (跨通道物理座標鎖定)

在圖層切換的瞬間，清空數學拓撲與運算狀態，但強制保留並重繪實體幾何邊界框，確保 $CH_1$ 與 $CH_2$ 在數學矩陣上的擷取位置達到像素級 ($1 \times 1 \ px$) 的絕對物理對齊。

10. Double Normalization Fold Change 雙重正規化生統架構

  • 內部對照組正規化：提取 Housekeeping Protein 同一泳道內總面積作為分母，消除轉漬效率誤差：
$$Normalized\_Val = \frac{Intra\_Base\_Value}{Control\_ROI\_Total\_Area}$$
  • 實驗對照組正規化：將特定泳道設為基準 1.0，執行第二次相除得出最終 Fold Change：
$$Final\_Ratio = \frac{Normalized\_Val_{Target\_Lane}}{Normalized\_Val_{Reference\_Lane}}$$

3. Operational Flow 操作流程
  • I/O, Identity & Multiplexing：支援多檔堆疊 (Multi-File Stacking) 與多通道 TIFF 匯入。系統為每個獨立檔案生成專屬 SHA-256 數位指紋確保絕對溯源。支援動態圖層切換以進行同座標擷取。
  • Spatial Definition：啟動裁切模式鎖定區域，調整傾角與微笑變形進行矩陣校正。
  • Analysis & Extraction：支援三軌並行。
    • 常規處理：點擊「▶ 𝓓ynamic Analysis」，動態微調背景半徑與 HDR 雙向滑桿尋找訊號
    • 自動模式：點擊「✨ Auto HDR」自動執行 MSER 降維掃描與收網
    • 局部打撈：開啟「🪄 Magic Wand」點擊極弱訊號，進行對數空間強制剝離
  • Data Locking：檢視紅色 Blob，點擊轉為藍色上鎖，系統於底層建立物理防禦牆 (masterLockMap) 防止水流溢流。
  • Biostats & Export：設定雙重正規化後將數據存入總表，並支援匯出 CSV、PNG、GLP PDF (含 Immutable Audit Trail) 及 加密簽章 .aq 專案檔。

4. Computational Pipeline 運算流程

系統於背景 Web Worker 中執行非同步運算，與 Main Thread 進行高度協作與平行化資料流處理。

  • A. Transient Decoding & Multiplex Queue 瞬態解碼與佇列 [Decode Worker]
    • 主執行緒建構非同步 Promise 佇列，依序將圖檔派發給動態生成的 Decode Worker。
    • 執行 Multiplex IFD 啟發式過濾，解碼後透過 Zero-Copy 傳遞 Float32 陣列。
    • 觸發 worker.terminate() 瞬間斬斷執行緒，強制 V8 引擎釋放所有解碼記憶體。
  • B. Memory Prep & Data Binding 記憶體準備與雙向通訊 [Main Thread]
    • 使用者切換通道或圈選 ROI 後，主執行緒提取目標子矩陣並傳送至 Math Worker。
    • 建立 Ping-Pong Buffers 準備承接即時拖曳與參數微調的高頻回傳結果。
  • C. Background & Geometry 幾何校正與背景處理 [Math Worker]
    • 執行 Quadratic Warp 矩陣重映射 (剛性旋轉與非對稱二階多項式微笑校正)
    • 執行 90% Robust Percentile 投影推算波段厚度，執行動態降採樣背景扣除
  • D. Topology & Cleaving 訊號拓撲掃描與降維切割
    • Mode 1: 常規動態微調 [Worker]
      • 執行 Hysteresis BFS 雙閾值水流蔓延，避開實體防禦牆
      • 遇到 Fusion Blob 觸發 1D Watershed 降維切割
      • 執行 Logarithmic Mapping，回傳陣列至 Main Thread
    • Mode 2: Magic Wand 局部對數空間 BFS [Main Thread]
      • 讀取 Ping-Pong Buffer，動態寬度 QC 基準萃取
      • 將局部線性強度映射至對數空間進行水流蔓延
      • 執行 1D Watershed + 強度對稱裁切，保留質心 Blob
    • Mode 3: Auto HDR (MSER) 異步協作 [Main Thread 狀態機 / Worker 迭代]
      • Zero-Latency Promise Hook： 摒棄傳統 setInterval 輪詢，主執行緒透過動態注入 window.autoHdrResolveHook 精準攔截 Worker 的 postMessage 回傳，實現 $\approx 0ms$ 延遲的極速雙向遞迴運算。
      • 掃描與狀態機評估 (Sweep & Evaluate)： 執行高密度水位下降掃描，於主執行緒維護各波段的生命週期狀態機 (State Machine)，動態評估「四大防護」與 $\Delta Area$ 寬容高原期，避免局部最佳解陷阱。
      • 觸底收網與 Phase 3 打撈： 掃描觸底後執行 End-of-Sweep Flush 強制收網，隨後啟動 Phase 3，利用成功波段的形態學 ($W_{ref}, H_{ref}$) 深入雜訊底層，以 $\rho=0.85$ 的高張力閾值進行強制剝離。
      • 幾何邊界淨化 (Geometric Crop)： 最終疊加交集對最小面積比 (IoM) 與 Bounding Crop 雙重過濾，精準刪除因非線性變形或剛性旋轉而超出原始 ROI 邊界的冗餘殘骸。
  • E. Assembly & Virtualization 組裝與虛擬化渲染
    • 執行交集對最小面積比 (IoM) 疊加過濾器剔除光暈殘骸
    • 動態泳道分群 (Dynamic Lane Grouping) 並寫入 imgCanvas
    • 觸發 Virtual DOM 表格重繪與雙重正規化生統運算
